# Security Policy

## Секреты

В репозиторий нельзя добавлять:

- `.env`;
- Telegram bot token;
- OpenAI API key;
- YooKassa keys;
- `bot.db`;
- логи;
- архивы с боевой базой.

Перед PR и релизом запускать:

```bash
make security-check
```

Проверка показывает файл, строку и тип риска, но не печатает секрет целиком.

Если секрет случайно попал в git:

1. Немедленно отозвать/перевыпустить ключ.
2. Сообщить владельцу проекта.
3. Не делать обычный revert как единственную меру: секрет уже мог попасть в историю.

## Админ-доступ

Админка доступна только Telegram ID из `ADMIN_IDS`.

В `production` `ADMIN_IDS` обязателен явно. `ADMIN_CHAT_ID` используется для уведомлений и обратной связи, но не должен быть единственным способом админ-доступа.

## Платежи

Plus нельзя выдавать вручную через обычный пользовательский сценарий.

Перед выдачей Plus платеж должен пройти проверку:

- статус YooKassa `succeeded`;
- `paid=true`;
- валюта `RUB`;
- ожидаемая сумма;
- `metadata.plan_code=plus`;
- `metadata.user_id`;
- `metadata.telegram_id`.

## Сообщение о проблемах

Для security-проблем использовать приватный канал владельца проекта или закрытый GitHub issue/обсуждение. Не публиковать токены, ключи, базы и приватные данные пользователей в открытых issues.
