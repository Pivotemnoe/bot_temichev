# TemichevVet - handoff для разработчика

Дата: 2026-06-02

## 1. Что это за проект

TemichevVet - Telegram-бот для владельцев кошек и собак.

Основные сценарии:

- онбординг и добавление питомца;
- карточка питомца;
- история здоровья;
- разбор жалобы и оценка срочности;
- контрольные вопросы после разбора;
- наблюдения, вес, напоминания, вакцинации;
- база знаний по питанию, уходу и вопросам;
- Plus-подписка;
- Telegram-админка и CSV-экспорт.

## 3. Основная структура

- `main.py` - точка входа, роутеры, фоновые воркеры.
- `app/config.py` - чтение `.env` и критичная конфигурация.
- `app/db.py` - SQLite-схема и функции работы с данными.
- `app/handlers/` - Telegram-обработчики.
- `app/pets_v2/` - карточка питомца, история, вес, вакцинации, напоминания.
- `app/services/` - selftest, аналитика, follow-up, paywall, knowledge, статические картинки.
- `app/payments/yookassa.py` - YooKassa-клиент и проверка платежа.
- `app/data/` - JSON-база питания, ухода и FAQ.
- `app/static/` - изображения и баннеры.
- `tools/` - локальные проверки.
- `Dockerfile` и `docker-compose.yml` - запуск через Docker.

## 4. Как запустить локально

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

После этого заполнить `.env` реальными значениями и запустить:

```bash
python main.py
```

При старте выполняется selftest: проверка env, базы и обязательных таблиц.

## 4.1. Как запустить через Docker

```bash
cp .env.example .env
docker compose up --build
```

В Docker база и логи пишутся в папку `data/`.

Папка `data/` не должна попадать в git.

## 5. Что нельзя коммитить

- `.env`;
- `bot.db`;
- реальные токены и ключи;
- логи;
- `.venv/`;
- `__pycache__/`, `*.pyc`;
- архивы с боевой базой;
- скриншоты с токенами или платежными ключами.

## 6. Как проверять изменения

Минимум перед коммитом:

```bash
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m compileall -q app tools main.py
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python - <<'PY'
from app.services.selftest import run_selftest
run_selftest()
print("selftest ok")
PY
git diff --check
```

Если изменение затрагивает питание/FAQ/уход:

```bash
.venv/bin/python tools/check_knowledge_json.py
```

Если изменение затрагивает Telegram UX, после локальных проверок запустить тестового бота и проверить сценарий вручную.

## 7. Git-процесс

Работать маленькими коммитами:

```bash
git status --short
git add <files>
git commit -m "Short clear message"
```

Не смешивать в одном коммите:

- платежи и тексты;
- админку и карточку питомца;
- рефакторинг и продуктовую правку;
- чистку файлов и изменение поведения.

## 9. Важные правила для следующего разработчика

- Не менять боевую версию на VPS без бэкапа.
- Не запускать новую версию на боевом токене, пока не проверен тестовый деплой.
- Не выдавать Plus без подтверждения платежа.
- Не добавлять админ-доступ через общий чат.
- В `production` обязательно нужен `ADMIN_IDS`.
- Все продуктовые изменения сначала проверять на тестовом Telegram-боте.
