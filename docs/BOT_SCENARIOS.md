# TemichevVet Bot Scenarios

Дата: 2026-06-03

Документ описывает пользовательские сценарии кнопка за кнопкой. Его нужно обновлять при каждом изменении меню, callback-кнопок или FSM-сценариев.

## 1. Первый вход

1. Пользователь отправляет `/start`.
2. Бот отправляет приветственный баннер.
3. Новый пользователь видит:
   - `👤 Зарегистрироваться`;
   - `⬅️ В меню`.
4. Если нажать `👤 Зарегистрироваться`:
   - бот спрашивает имя;
   - пользователь вводит имя;
   - бот спрашивает тип питомца;
   - пользователь выбирает `🐱 Кот/Кошка` или `🐶 Собака`;
   - бот спрашивает кличку;
   - пользователь вводит кличку или `Пропустить`;
   - бот создаёт пользователя и питомца;
   - бот показывает главное меню и онбординг.
5. Если нажать `⬅️ В меню`:
   - бот открывает главное меню без регистрации;
   - для разборов, истории и напоминаний всё равно потребуется питомец.

## 2. Повторный `/start`

1. Пользователь отправляет `/start`.
2. Бот отправляет баннер.
3. Бот показывает:
   - краткое приветствие;
   - основные действия;
   - главное меню.
4. Если у пользователя есть онбординг, который ещё нужно показать, он показывается после приветствия.

## 3. Главное меню

Основные кнопки:

- `🩺 Разобрать жалобу`;
- `📜 История здоровья`;
- `📊 Наблюдения`;
- `🍽️ Питание`;
- `🧴 Уход и привычки`;
- `❓ Вопросы и ответы`;
- `⏰ Напоминания`;
- `🐾 Мои животные`;
- `👤 Моя подписка`;
- `ℹ️ О боте`;
- `✉️ Обратная связь`.

Универсальный выход из сценариев:

- `/cancel`;
- `⬅️ В меню`.

## 4. Разбор жалобы

1. Главное меню -> `🩺 Разобрать жалобу`.
2. Если питомцев нет:
   - бот предлагает добавить питомца.
3. Если питомец один:
   - бот сразу переходит к возрасту.
4. Если питомцев несколько:
   - бот показывает список питомцев;
   - пользователь выбирает питомца.
5. Бот спрашивает возрастную группу.
6. Пользователь выбирает возраст или вводит текстом.
7. Бот спрашивает длительность проблемы.
8. Пользователь выбирает длительность или вводит текстом.
9. Бот просит описать жалобу.
10. Пользователь пишет симптомы.
11. До LLM бот проверяет красные симптомы:
    - судороги;
    - тяжёлое дыхание/одышка;
    - кровь/кровотечение;
    - отравление;
    - потеря сознания.
12. Если красные симптомы есть:
    - бот сразу пишет `🟥 Срочно в клинику`;
    - не ждёт LLM;
    - не списывает лимит;
    - сохраняет событие в историю питомца.
13. Если красных симптомов нет:
    - бот списывает лимит;
    - вызывает LLM;
    - показывает ответ;
    - сохраняет разбор в историю и наблюдения.
14. После сценария бот показывает `Что дальше?`:
    - `🐾 Открыть карточку питомца`;
    - `➕ Добавить напоминание`;
    - `🩺 Новый разбор`;
    - `⬅️ В меню`.

## 5. Карточка питомца

1. Главное меню -> `🐾 Мои животные`.
2. Бот показывает список питомцев.
3. Пользователь выбирает питомца.
4. Бот показывает карточку:
   - данные питомца;
   - `Что важно сейчас`;
   - статусы;
   - последние разборы.
5. Кнопки карточки:
   - `📌 Обзор`;
   - `⭐ Сделать основным` или `⭐ Снять основной`;
   - `💉 Вакцинации`;
   - `⏰ Напоминания`;
   - `📊 Наблюдения`;
   - `⚖️ Вес`;
   - `📜 Вся история`;
   - `🩺 Разобрать жалобу`;
   - `✏️ Изменить`;
   - `🗑 Удалить`;
   - `⬅️ В меню`.

## 6. Напоминания из главного меню

1. Главное меню -> `⏰ Напоминания`.
2. Бот открывает раздел "Напоминания и график".
3. Кнопки:
   - `➕ Добавить напоминание`;
   - `📋 Мои напоминания`;
   - `⬅️ В меню`.
4. Добавление:
   - выбор питомца;
   - выбор типа;
   - дата;
   - время или `Пропустить`;
   - периодичность;
   - сохранение.
5. Шаблоны:
   - `💉 Вакцинация`;
   - `🪳 Обработка от паразитов`;
   - `🩺 Плановый осмотр`;
   - `✂️ Груминг`;
   - `🍽️ Корм/диета`;
   - `📌 Другое`.

## 7. Напоминания из карточки питомца

1. Карточка питомца -> `⏰ Напоминания`.
2. Бот показывает только напоминания выбранного питомца.
3. Кнопки:
   - `➕ Добавить напоминание`;
   - `✏️ Изменить напоминание`, если есть активные;
   - `🗑 Удалить напоминание`, если есть активные;
   - `⬅️ Назад`.
4. Добавление из карточки:
   - выбор шаблона или `✍️ Свой текст`;
   - дата;
   - время или `-`;
   - периодичность;
   - заметка или `-`;
   - сохранение;
   - `Что дальше?`.

## 8. Вес

1. Карточка питомца -> `⚖️ Вес`.
2. Бот показывает последний вес и историю.
3. Кнопки:
   - `➕ Добавить вес`;
   - `📋 История веса`;
   - `⬅️ Назад`;
   - `⬅️ В меню`.
4. Добавление веса:
   - пользователь вводит число в кг;
   - добавляет заметку или пропускает;
   - подтверждает сохранение;
   - бот сохраняет вес и показывает `Что дальше?`.

## 9. Питание

1. Главное меню -> `🍽️ Питание`.
2. Бот показывает меню питания.
3. Кнопки:
   - `🔍 Найти продукт`;
   - `✅ Что можно`;
   - `⛔ Что нельзя`;
   - `⬅️ В меню`.
4. Пользователь вводит продукт.
5. Если это готовое блюдо, например `борщ`, `гуляш`, `котлета`, `паста`:
   - бот просит состав через запятую;
   - пользователь вводит ингредиенты;
   - бот проверяет состав и опасные добавки.
6. После ответа бот показывает `Что дальше?`.

## 10. Подписка и оплата

1. Главное меню -> `👤 Моя подписка`.
2. Бот показывает тариф и лимиты.
3. Пользователь нажимает `⭐ Перейти на Plus` или оплату Plus.
4. Бот создаёт YooKassa-платёж.
5. Бот показывает:
   - `💳 Открыть оплату`;
   - `✅ Я оплатил (проверить)`;
   - `⬅️ Назад`.
6. Проверка оплаты:
   - бот берёт последний платёж текущего пользователя;
   - запрашивает статус YooKassa;
   - проверяет статус, сумму, валюту, `paid=true`, `metadata.user_id`, `metadata.telegram_id`, `metadata.plan_code`;
   - активирует Plus только после успешной проверки.

## 11. Админка

1. Админ пишет `админ` или `/admin`.
2. Бот проверяет Telegram ID по `ADMIN_IDS`.
3. Не-админ получает отказ, попытка логируется.
4. Админ видит панели:
   - отчёт за период;
   - воронка;
   - подписки;
   - retention;
   - расходы;
   - источники;
   - экспорт CSV;
   - платежи;
   - ручная выдача/снятие Plus.

## 12. Обратная связь

1. Главное меню -> `✉️ Обратная связь`.
2. Бот просит текст сообщения.
3. Пользователь отправляет сообщение.
4. Бот сохраняет feedback и отправляет уведомление в `FEEDBACK_CHAT_ID` или `ADMIN_CHAT_ID`.

## 13. Правила для новых сценариев

1. Любой новый FSM-сценарий должен принимать `/cancel`.
2. Любой callback с `pet_id`, `reminder_id`, `payment_id` обязан заново проверять владельца.
3. После завершения сценария желательно показывать `Что дальше?`.
4. Для медицинских сценариев нельзя ставить диагноз, назначать лекарства, дозировки или схемы лечения.
