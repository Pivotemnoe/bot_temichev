# TemichevVet - план защиты админки и платежей

Дата: 2026-06-01

## 1. Админ-панель

Текущее решение:

- доступ только по Telegram ID из `ADMIN_IDS`;
- в `production` `ADMIN_IDS` должен быть задан явно;
- `ADMIN_CHAT_ID` используется для уведомлений и обратной связи, но не как неявный админ-доступ в `production`;
- чужой пользователь, который напишет `админ`, не получает админку;
- неавторизованная попытка логируется;
- администратору отправляется уведомление о попытке доступа с ограничением частоты уведомлений.

Что важно на VPS:

- в `.env` обязательно указать `ENV=production`;
- в `.env` обязательно указать `ADMIN_IDS=<telegram_id_админа>`;
- не давать доступ через общий чат;
- не пересылать `.env` в открытом виде;
- при смене администратора менять `ADMIN_IDS` и перезапускать сервис.

## 2. Платежи

Текущее решение:

- YooKassa-ключи берутся только из `.env`;
- selftest останавливает запуск, если задан только один из ключей YooKassa;
- `YOOKASSA_RETURN_URL` должен быть HTTPS;
- Plus активируется только после запроса статуса платежа у YooKassa;
- перед выдачей Plus проверяется:
  - статус `succeeded`;
  - `paid=true`;
  - валюта `RUB`;
  - сумма тарифа;
  - `metadata.plan_code=plus`;
  - `metadata.user_id`;
  - `metadata.telegram_id`.

Что ещё нужно перед боевым запуском:

- проверить платежи на VPS с тестовыми ключами;
- проверить успешную оплату;
- проверить отмену оплаты;
- проверить повторное нажатие оплаты;
- проверить, что Plus не выдается при несовпадении суммы или пользователя;
- после тестов подключать боевые ключи.

## 3. Секреты

Правила:

- `.env` не коммитить;
- `bot.db` не коммитить;
- токены Telegram и YooKassa не писать в отчеты и логи;
- перед передачей архива проверять, что внутри нет `.env`, базы и логов;
- для VPS делать резервную копию старой версии до любых изменений.

## 4. Следующие улучшения

- добавить админ-команду проверки пользователя по Telegram ID;

- добавить отдельный журнал security-событий в БД;
