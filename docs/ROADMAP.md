# TemichevVet Roadmap

Дата: 2026-06-03

Этот документ фиксирует приоритеты разработки после текущего рабочего состояния. Он нужен, чтобы следующий разработчик быстро понял, что делать дальше и что нельзя трогать без отдельной проверки.

## P0 - перед боевым деплоем

1. Проверить `.env` для production:
   - `BOT_TOKEN` боевого бота;
   - `OPENAI_API_KEY`;
   - `ADMIN_IDS`;
   - `ADMIN_CHAT_ID`;
   - `YOOKASSA_SHOP_ID`;
   - `YOOKASSA_SECRET_KEY`;
   - `YOOKASSA_RETURN_URL`;
   - `ENV=production`.
2. Сделать backup текущей VPS-базы до любых действий:
   - локально: `make backup-db`;
   - VPS: см. `docs/BACKUP_RESTORE.md`.
3. Проверить, что на VPS запущен один процесс с одним `BOT_TOKEN`.
4. Проверить платежи YooKassa на тестовом сценарии:
   - создание платежа;
   - проверка `✅ Я оплатил`;
   - запись в `payments`;
   - активация Plus на 30 дней;
   - отсутствие повторного продления при повторной проверке одного платежа.
5. Проверить security:
   - `make check`;
   - `make security-check`;
   - `make medical-safety-check`;
   - ручная проверка, что не-админ не открывает админку.

## P1 - безопасность и надёжность

1. Расширить callback-access тесты при добавлении каждого нового callback:
   - `pet_id` проверяется через владельца;
   - `reminder_id` проверяется через владельца и питомца;
   - платеж проверяется через текущего пользователя и metadata YooKassa.
2. Добавить мониторинг ошибок:
   - ежедневный просмотр `errors.log`;
   - отдельный алерт в `ADMIN_CHAT_ID` при падении фоновых воркеров.
3. Добавить контроль миграций БД:
   - отдельный файл с версиями схемы;
   - проверка несовместимых изменений перед запуском.
4. Расширить red-flag список:
   - травма/падение с высоты;
   - вздутие живота у собаки;
   - многократная рвота;
   - невозможность мочиться;
   - высокая температура, если пользователь указал число.

## P2 - продукт и UX

1. Улучшить первый экран:
   - короткая инструкция;
   - что бот делает;
   - что бот не делает;
   - когда срочно в клинику.
2. Доработать питание:
   - больше бытовых блюд;
   - больше синонимов ингредиентов;
   - отдельная логика для щенков/котят и пожилых питомцев.
3. Улучшить карточку питомца:
   - компактный блок "что важно сейчас";
   - ближайшее напоминание;
   - последние красные/жёлтые события;
   - динамика веса.
4. Улучшить "Что дальше" после каждого сценария:
   - открыть карточку;
   - добавить напоминание;
   - начать новый разбор;
   - вернуться в меню.

## P3 - документация и команда

1. Поддерживать актуальными:
   - `docs/BOT_SCENARIOS.md`;
   - `docs/DB_SCHEMA.md`;
   - `docs/SECURITY_CHECKLIST.md`;
   - `docs/RELEASE_CHECKLIST.md`.
2. Для каждого PR указывать:
   - что изменено;
   - какие проверки запускались;
   - какие Telegram-сценарии нужно проверить вручную.
3. Добавить GitHub Actions CI после проверки, что GitHub-токен имеет `workflow` scope.

## Отложено

1. Блок клиник как отдельный B2B-продукт.
2. Android-приложение.
3. Полноценная CRM для клиник.
4. Автоматические платежи/рекуррентные подписки.

## Правило изменения приоритетов

Если новая задача влияет на платежи, безопасность, доступ к чужим данным или медицинскую срочность, она автоматически поднимается минимум до P1.
