# TemichevVet - security checklist

Дата: 2026-06-03

## Перед каждым PR

- [ ] `make check` проходит.
- [ ] `make security-check` проходит.
- [ ] `git diff --check` проходит.
- [ ] В diff нет `.env`, `bot.db`, `data/`, логов, архивов и скриншотов с ключами.
- [ ] Новые callback-обработчики проверяют текущего пользователя и владельца данных.
- [ ] Новые админ-кнопки начинаются с `admin:` и проходят проверку `ADMIN_IDS`.
- [ ] Новые админ-действия пишутся в `admin_audit_log`.
- [ ] Новые платежные действия не выдают Plus без серверной проверки.
- [ ] Новые пользовательские сценарии с риском спама учитываются в rate limit.

## Перед релизом

- [ ] `ENV=production` задан только на сервере.
- [ ] `ADMIN_IDS` заполнен явным Telegram ID администратора.
- [ ] `ADMIN_CHAT_ID` используется только для уведомлений.
- [ ] `.env` не отслеживается git.
- [ ] `bot.db` и `data/` не отслеживаются git.
- [ ] YooKassa-ключи не лежат в git.
- [ ] Проверена неуспешная оплата: Plus не выдаётся.
- [ ] Проверена успешная оплата: Plus выдаётся только текущему пользователю.
- [ ] Админский экран `⚙️ Статус` не выводит токены, ключи и секретные значения.
- [ ] Админский экран `🛡 Аудит` показывает последние действия и отказанные попытки.
- [ ] Есть бэкап текущего кода, `.env` и базы.
- [ ] Есть понятный план отката.

## GitHub

- [ ] `main` защищён branch protection.
- [ ] Прямой push в `main` разрешён только владельцу или полностью отключён.
- [ ] Включён review перед merge.
- [ ] Включён secret scanning, если тариф GitHub позволяет.
- [ ] Включена 2FA у владельца и разработчиков с write/admin доступом.
- [ ] Разработчикам выданы минимальные права.
- [ ] GitHub Actions добавляется только после появления token с `workflow` scope.

## VPS

- [ ] На один `BOT_TOKEN` запущен только один процесс.
- [ ] `.env` не доступен публично и не лежит в web-root.
- [ ] Права на `.env` ограничены пользователем сервиса.
- [ ] Бэкапы базы не лежат в репозитории.
- [ ] Логи не содержат токены, YooKassa secret, OpenAI key и персональные дампы.
- [ ] После смены `.env` сервис перезапущен.

## Инцидент с секретом

1. Немедленно отозвать или перевыпустить ключ.
2. Остановить старый процесс, если ключ относится к Telegram/VPS.
3. Обновить `.env` на сервере и локально.
4. Перезапустить сервис.
5. Проверить `/start`, админку и оплату.
6. Очистить git-историю только как дополнительную меру. Revert не отменяет утечку.

## Инцидент с платежом

1. Проверить запись в таблице `payments`.
2. Проверить статус платежа в YooKassa.
3. Проверить сумму, валюту, `metadata.user_id`, `metadata.telegram_id`.
4. Если Plus выдан ошибочно, снять доступ вручную через базу или отдельную админ-команду.
5. Сохранить логи инцидента без секретов.

## Инцидент с доступом к чужим данным

1. Остановить релиз или откатиться на предыдущую версию.
2. Найти обработчик, где не проверяется `owner_id`.
3. Добавить проверку через `get_pet_for_user(owner_id, pet_id)` или аналогичный owner-check.
4. Проверить вручную поддельный callback/ID.
5. Добавить пункт в release checklist, если это новый тип данных.
